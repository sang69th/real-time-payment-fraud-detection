# Problem Statement

This project builds a payment fraud detection platform (eg: for PayPal, Revolut, GooglePay)

The goal is to identify suspicious transactions using historical payment data.

The system will:
- analyze transaction patterns
- train a fraud detection model
- predict whether a new transaction is suspicious
- show fraud insights in a dashboard

Target users:
- fraud analysts
- risk teams
- operations teams
- Risk Analysts

Business value:
- reduce financial loss
- detect suspicious transactions faster
- support better fraud monitoring

Consider a scenario where a user makes 30 transactions from an account in a span of 5 minutues. This raises a suspicion that the account could be hacked or could raise a flag to the system for the risk analysts to have a look at the transactions.
